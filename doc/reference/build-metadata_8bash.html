Detailed Description

Documents and reports metadata embedded in the generated artifact.

The Phase 1 build process writes BOOTSTRAP_VERSION, BOOTSTRAP_BUILD_DATE, and BOOTSTRAP_BUILD_COMMIT into dist/bootstrap.bash before this source fragment is concatenated. The default assignments below keep this module safe to analyze, source, or execute outside the generated artifact.

Metadata is informational. It helps users and contributors understand which source revision produced an artifact, but it does not control package installation, planning, configuration, or command-line behavior.

◆ bootstrap_print_version()

bootstrap_print_version ( )

Prints public version information for the generated artifact.

The version output is intentionally short, stable, and easy to read from both interactive terminals and automated logs. It exposes build metadata without requiring users to inspect the generated script manually.

The build metadata is embedded by the Makefile when dist/bootstrap.bash is generated. Local source execution uses the safe defaults above, which makes this function predictable even outside the generated artifact.
