Detailed Description

Provides privilege escalation helpers for execution backends.

Package managers often require elevated privileges, but backend modules should not need to decide whether to run directly, use sudo, use doas, or fail. This runtime module keeps that policy in one place.

Executors pass a command and its arguments to bootstrap_privilege_run(). This module then chooses the safest available way to run that command for the current process.

bootstrap_privilege_command_prefix ( )

Selects the command prefix needed for privileged execution.

If the current process is already running as root, no prefix is needed. When running as a non-root user, sudo is preferred when available, followed by doas. If neither tool is available, execution fails conservatively.

Standard Output
Optional privilege command prefix.
Return values
0A usable privilege strategy was selected.
71No usable privilege escalation command was available.
Examples
prefix="$(bootstrap_privilege_command_prefix)" || exit "$?"
printf 'privilege prefix: %s\n' "${prefix:-none}"

bootstrap_privilege_effective_uid ( )

Prints the current effective user identifier.

This helper exists so tests may override it after sourcing the generated artifact. Production code should treat it as a thin wrapper around id -u.

Standard Output
The effective user identifier.
Return values
0The user identifier was printed successfully.
Examples
uid="$(bootstrap_privilege_effective_uid)"
printf 'effective uid: %s\n' "${uid}"
